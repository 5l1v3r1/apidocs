    Welcome to the Bitwyre API!

    All API URLs should use the domain https://api.bitwyre.com

    We have Public and Private API. Public endpoints can use either GET or POST. Private endpoints must use POST and be set up using the Authentication schema desribed below.

    All endpoints returns either a JSON object or array along with some error messages (if any).

    Public API

    Authentication

    To authorize, use something like this:

    curl "https://api.bitwyre.com"
  -H "API-Key: someapikey"
  -H "API-Sign: somesignature"
  -d "param1=value1&param2=value2"
  -X POST

    Make sure to replace someapikey with your api-key.

    Bitwyre uses HTTP Header authentication to allow access to the API. You can get a new Bitwyre API key pairs (api-key and secret-key) at our developer portal.

    All Private API requests must be done using POST and Bitwyre expects for the following headers to be included in all API requests to the server:

    • API-Key: someapikey
    • API-Sign: signature

    Message signature is created using HMAC-SHA512 of (URI path + SHA256(nonce + POST data)) and secret-key. POST data is something like param1=value1&param2=value2.

    Replace someapikey with your `api-key`.

    Opening a New Order

    curl "https://api.bitwyre.com/private/order/create"
  -H "API-Key: someapikey"
  -H "API-Sign: somesignature"
  -d "pair=btcidr&type=buy&ordertype=limit&price=125000000&volume=500000&test=false"
  -X POST

    Response

    {
  "error": [],
  "result": {
    "validation": "ok",
    "order_id": ["1234"]
  }
}

    Create a new trade order.

    Endpoint

    https://api.bitwyre.com/private/order/create

    Parameters

    Parameter Required Value Description
    pair true btcidr, ethidr, etc. Pair to open new order on.
    type true buy/sell Type of order. Buying against or selling against.
    ordertype true market/limit Market price order or limit price order.
    price false numerical Depends upon ordertype. Only on market is optional, otherwise required. In either decimal or double.
    volume true numerical Order volume in either decimal or double.
    test true true/false If true, validate inputs only. Order is not submitted and no order_id will be returned.

    Errors

    The Bitwyre API uses the following HTTP error codes:

    Code Meaning Description
    400 Bad Request Your request is invalid.
    401 Unauthorized Your API key is wrong.
    403 Forbidden Oops. You're not allowed to access that.
    404 Not Found Whatever you're looking for could not be found.
    405 Method Not Allowed You tried to access stuffs with an invalid method.
    406 Not Acceptable You requested a format that isn't processable.
    410 Gone The things you requested has been removed from our servers.
    418 I'm a teapot. We refuse to brew your coffee because you've been banned.
    429 Too Many Requests You're calling in too fast! Slow down!
    500 Internal Server Error We had a problem on our side. Try again later. Sorry!
    503 Service Unavailable We're temporarily offline for maintenance. Please try again later.
